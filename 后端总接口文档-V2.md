> 本文档基于后端已实现接口与规划接口输出，对应前端 PyrographyArt（Vue 3 + TypeScript + Element Plus + Pinia）。
>
> 本版本覆盖全部模块：**认证、用户资料、文件上传、互动帖子、评论、AI 智能问答、传世名作、收藏夹、传承基地、展览活动、文创商城、捐赠支持、共创计划、消息通知、线上轻互动**。
>
> V2 是一次全面的**代码规范升级**——统一用大写英文枚举+业务前缀，统一字段命名风格（驼峰+语义化），同时补充了一些之前缺失的字段。后续开发应该以 V2 为准

## 1. 通用约定

### 1.1 Base URL

- **本地开发**：`/burned-foil/api`（Vite 代理转发）
- **生产环境**：`http://42.194.230.173:6666/burned-foil/api`

### 1.2 鉴权方式

后端鉴权统一为 **JWT**。

默认请求头约定：

- `Authorization: Bearer <token>`（登录后获取）
- `User-ID: <userId>`（评论模块必填）

说明：

- 公开接口 `GET /api/posts`、`GET /api/comments/post/{postId}` 等不需要登录。
- 认证接口 `POST /api/auth/login`、`POST /api/auth/register` 不需要登录。
- 需要登录的接口标注为"需要登录"或"JWT"。
- 评论模块使用 `User-ID` 头标识用户身份。

### 1.3 统一响应格式

所有接口统一返回：

```json
{
  "code": 0,
  "msg": "成功",
  "data": {},
  "success": true,
  "trace_id": "8f45a7d5d8b24d7a"
}
```

字段说明：

- `code`：业务状态码，`0` 表示成功，非 0 表示异常
- `msg`：业务提示信息（前端可直接展示）
- `data`：返回数据，可能为对象、数组或 `null`
- `success`：`true` 表示成功，`false` 表示业务异常
- `trace_id`：链路追踪 ID，便于排查问题

### 1.4 分页响应格式

分页接口的 `data` 统一为：

```json
{
  "list": [],
  "total": 100,
  "page": 1,
  "pageSize": 20
}
```

### 1.5 时间约定

- 时间字段使用 ISO 8601：`2025-01-01T10:00:00Z` 或 `2025-01-01T10:00:00`
- 评论模块使用 `yyyy-MM-dd HH:mm:ss` 格式
- 分页参数：
  - `page`：页码，从 `1` 开始
  - `pageSize`：每页条数，默认 `20`

### 1.6 请求参数示例写法

本文档中的"请求参数示例"统一采用如下结构：

- `path`：路径参数
- `query`：查询参数
- `body`：JSON 请求体
- `formData`：`multipart/form-data` 表单字段

无对应参数时，会明确写为 `null` 或空对象 `{}`。

### 1.7 通用枚举约定

#### 1.7.1 用户角色
